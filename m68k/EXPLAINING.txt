_start: 
    Точка входа в программу. Устанавливает указатель стека (4092) для безопасного вызова вложенных процедур и вызывает главную функцию main.

main: 
    "Настройка курсоров". 
    Программа загружает базовые адреса в регистры: порты ввода (128) и вывода (132), буфер строки формата (2048) и буфер итогового результата (2500).

init_pow_loop: 
    Заполняет таблицу степеней (3000). 
    Генерирует множители (10, 100, 1000 и т.д.) путем умножения на 10. 
    Числа складываются в память задом наперед, чтобы потом использовать их для конвертации числа в строку.

read_fmt: 
    Читает символы напрямую из порта ввода (128) и записывает их в буфер формата (2048), пока не встретит перенос строки (\n). 
    Ограничивает длину 33 символами для защиты от переполнения.

fmt_done: 
    Закрывает строку в буфере 2048 нуль-терминатором (0), чтобы процессор знал, где конец текста.

scan_loop: 
    Главный цикл сканирования. 
    Читает из буфера формата (2048). 
    Если видит обычный символ — копирует его в итоговый буфер (2500). 
    Если видит % — понимает, что начался формат, и уходит на handle_format.

handle_format: 
    Очищает ячейку флага выравнивания (3600). 
    Если сразу после % идет минус (например, %-5d), записывает в этот флаг 1 (выравнивание влево).

hf_loop: 
    Собирает ширину поля. 
    Превращает ASCII-символы ширины (например, 5 и 2 из %52d) в реальное число, умножая накопленный результат на 10. 
    Цикл прерывается, когда встречает букву d.

print_integer: 
    Управляющая метка для вывода числа. 
    Запускает чтение вводимого числа (read_int). 
    Проверяет знак: если число отрицательное, пишет минус во временный буфер (3500) и делает число положительным. 
    Также перехватывает краевой случай (2 миллиарда) для предотвращения аппаратного переполнения.

pi_buf_pos / pi_buf_loop / pi_buf_sub: 
    Ядро перевода числа в текст. 
    Процессор берет степени десятки из таблицы (3000) и вычитает их из числа, подсчитывая количество успешных вычитаний. 
    Это количество и становится цифрой текущего разряда.

pi_buf_digit_done / pi_buf_write / pi_buf_next: 
    Блок отсечения ведущих нулей. 
    Если цифра значащая, она переводится обратно в ASCII (+48) и дописывается во временный буфер (3500).

pi_check_align: 
    Вычисляет количество нужных пробелов (ширина из формата минус длина получившегося числа). 
    Затем проверяет флаг выравнивания (3600) для выбора направления.

pi_right_align / pi_ra_pad_loop / pi_ra_num: 
    Выравнивание вправо. 
    Сначала генерирует цикл пробелов в итоговый буфер (2500), а затем переписывает туда само число из временного буфера (3500).

pi_left_align / pi_la_num_loop / pi_la_pad: 
    Выравнивание влево. 
    Наоборот: сначала переносит цифры из буфера 3500 в буфер 2500, а затем добивает оставшееся пространство пробелами.

read_int / ri_check_digit / ri_loop: 
    Читает само число-аргумент из порта ввода (128). 
    Извлекает символы, проверяет, что это цифры (от '0' до '9'), и собирает из них 32-битное значение, сдвигая разряды умножением на 10.

ri_check_exact / ri_check_neg_overflow: 
    Защитный блок. 
    Строго контролирует, чтобы число не превысило 32-битные лимиты (2147483647 и -2147483648) на последней вводимой цифре. 
    Если лимит превышен — выбрасывает ошибку.

ri_invalid / ri_done / ri_exit: 
    Завершает чтение, при необходимости делает число отрицательным и выходит из процедуры через rts.

program_exit / pe_loop / pe_done: 
    Финальный аккорд. 
    Программа ставит нулевой байт в конец собранного итогового буфера (2500). 
    Затем в цикле побайтово переписывает всю сформированную строку из этого буфера прямиком в порт вывода (132).

error_exit: 
    Универсальный обработчик крашей. 
    Если формат неверный, буфер переполнен или число слишком большое, пишет -1 в порт вывода (132) и намертво останавливает процессор (halt).
