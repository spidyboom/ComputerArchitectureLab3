4. Конвенции вызова (Calling Conventions / ABI)
Конвенция — это «джентльменское соглашение» между программистами (или компиляторами) о том, как распределять обязанности внутри процессора, чтобы код разных людей мог работать вместе без сбоев.

Распределение обязанностей (Caller-saved vs Callee-saved)
Как мы уже разбирали, регистры делятся на две категории. Представь, что Функция А вызывает Функцию Б:
Caller-saved (ra, t0-t6, a0-a7): Функция А должна сама спасать эти регистры в стек, если они понадобятся ей после возврата из Функции Б. Функция Б имеет право стереть их в ноль без предупреждения.
Callee-saved (sp, s0-s11): Функция Б обязана уважать эти регистры. Если они ей нужны, она сохраняет их в стек при входе и восстанавливает при выходе. Для Функции А они гарантированно останутся неизменными.

Правила работы со стеком
Выравнивание (Alignment): Согласно официальной конвенции RISC-V, указатель стека sp всегда должен быть выровнен по границе 16 байт. Даже если тебе нужно сохранить всего один 4-байтный регистр, ты обязан уменьшить sp минимум на 16 (например, addi sp, sp, -16). В учебных симуляторах этим часто пренебрегают ради экономии места, но на реальном железе нарушение выравнивания стека приведет к аварийной остановке.
Направление: Стек строго растет вниз (к меньшим адресам).

5. Конвейер процессора (Pipelining)
Конвейер — это способ ускорения процессора за счет параллельного выполнения разных этапов нескольких инструкций. Это работает как автоконвейер: пока одна машина красится, следующая собирается, а третья только варится.

В классическом RISC-V используется 5-ступенчатый конвейер:
[Инструкция 1] -> Fetch -> Decode -> Execute -> Memory -> WriteBack
[Инструкция 2] ->          Fetch  -> Decode  -> Execute -> Memory -> WriteBack
[Инструкция 3] ->                    Fetch   -> Decode  -> Execute -> Memory

IF (Instruction Fetch): Процессор берет команду из памяти кода по адресу в PC.

ID (Instruction Decode): Команда расшифровывается, извлекаются нужные регистры.

EX (Execute): Арифметико-логическое устройство (АЛУ) считает результат (например, add или addi).

MEM (Memory Access): Происходит чтение/запись в ОЗУ (работает только для команд lw/sw).

WB (Write Back): Финальный результат записывается обратно в регистровый файл (x1-x31).

6. Ключевые особенности архитектуры RISC-V
Одинаковый размер команд: Все базовые инструкции занимают ровно 32 бита (4 байта). Это позволяет невероятно легко рассчитывать адреса в памяти и упрощает этап IF (Fetch) на конвейере.

Архитектура Load-Store: Ни одна вычислительная команда не может работать с оперативной памятью. Память — отдельно (lw/sw), вычисления — отдельно в регистрах (add, sub).

Регистр zero (x0): Аппаратный ноль — гениальное решение, убирающее необходимость в десятках специализированных команд (обнуление, копирование регистра, знаковые инверсии делаются через обычное сложение с zero).

Отсутствие аппаратного регистра флагов: В RISC-V нет регистра состояний вроде EFLAGS. Команды условного перехода (beq, bne, blt) сами сравнивают значения регистров «на лету» на этапе EX. Это убирает зависимость между инструкцией сравнения и инструкцией перехода, разгружая конвейер.

Отказ от сложных режимов адресации: В отличие от x86, где можно написать команду с автоматическим умножением индекса и сдвигом, в RISC-V доступен только один режим адресации памяти: смещение(базовый_регистр). Если нужно посчитать сложный адрес — считай его сам в регистрах с помощью команд add и slli.



sll / srl — Логические сдвиги влево и вправо (освобождающиеся биты забиваются нулями).
sra — Арифметический сдвиг вправо (сохраняет знаковый бит — если число было отрицательным, слева будут вставляться единицы, а не нули).
and / or / xor — Побитовые логические операции.
slt / sltu — Set Less Than (знаковое и беззнаковое). Если первый регистр меньше второго, в результирующий регистр записывается 1, иначе 0.

addi / andi / ori / xori — Сложение и логика с константой. Особенность: размер константы ограничен 12 битами со знаком (от -2048 до +2047).
slli / srli / srai — Сдвиги на фиксированное число бит. Особенность: константа здесь занимает всего 5 бит (так как сдвинуть 32-битный регистр можно максимум на 31 позицию).
slti / sltiu — Сравнение регистра с 12-битной константой.

lw (Load Word) — Загружает полноценное 32-битное слово (4 байта).
lh / lb — Загружает полуслово (2 байта) или байт. Особенность: процессор автоматически растягивает этот байт до 32 бит, заполняя левую часть его знаковым битом.
lhu / lbu — Загружает полуслово/байт без знака. Левая часть 32-битного регистра просто забивается нулями.

sw (Store Word) — Записывает 4 байта из регистра в память.
sh / sb — Записывает в память только младшие 2 байта или 1 байт из регистра.
